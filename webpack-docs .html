World 01 — Entry Points

Where every
bundle begins

Webpack starts from one or more entry files and traces every import and require() outward, building a complete dependency graph before writing a single byte to disk.

// webpack.config.js
module.exports = {
  entry: {
    main: './src/index.js',
    admin: './src/admin.js'
  }
}

entry — The point or points where to start the application bundling process. If an array is passed, all items will be processed.

Supports: String · Array · Object · Function

Webpack 6 adds support for HTML entry points, allowing './src/index.html' as entry.

// Dynamic entry (function)
entry: () => new Promise(
  resolve => resolve({
    a: './src/a.js',
    b: './src/b.js'
  })
)

World 02 — Module Graph

Connected nodes,
infinite power

Every file in your project is a module. Webpack 6 builds an enhanced dependency graph that understands async chunks, dynamic imports, and circular dependencies with millisecond precision.

Module types supported by Webpack 6:

• ESM — import / export
• CommonJS — require()
• CSS/SCSS — via loaders
• Assets — images, fonts, SVG
• WASM — WebAssembly modules

module: {
  rules: [
    { test: /\.css$/,
      use: ['style-loader', 'css-loader'] }
  ]
}

World 03 — Code Splitting

Divide to
conquer load time

    /* webpackChunkName: "page" */

  );
  Page.render();
};
